Notion Automations

4 automation rules to reduce manual checking

These Notion database automations bridge the alerting gap. Instead of ops manually checking the Task Dashboard every few minutes, automations push notifications when members take actions. Set these up in each database's automation panel.

Setup Instructions
01Open each database (Components, Component Comments, Trips) in Notion
02Click the lightning bolt icon (Automations) in the database toolbar
03Click "New automation" and configure the trigger and action per the table above
04For Slack integration: use Notion's built-in "Send Slack notification" action block
05Test each automation by creating a sample record and verifying the task/notification appears
Limitations
  • Notion automations cannot enforce forward-only state transitions -- ops can still set any status
  • Slack notifications are plain text only -- no interactive buttons or approval flows
  • Task creation adds records but does not assign or prioritize automatically
  • Automation 4 (suggest advancing) requires a formula field to check "all components booked" -- complex to maintain
