Performance

Performance targets and offline caching strategy

P95 performance targets for all major operations, plus offline caching strategy with pre-cache triggers, storage limits, and conflict resolution approach.

Performance Targets

Trip page initial load (4G) < 2 sec (P95)
Component detail fetch < 500ms (P95)
Decision PATCH response < 1 sec (P95)
Comment POST response < 1 sec (P95)
Image carousel first load (4G) < 1 sec (P95)

Offline Caching Strategy

Pre-cache Trigger

Full trip data pre-cached when status reaches Booking Complete

Storage Limits

~50MB per trip, 200MB total device cap

Offline Indicator

Warm gray banner + queued actions counter badge

Conflict Resolution

Server-wins on reconnect. Queued actions replayed in order.

Polling Strategy

V1 (Ship)

60-second polling interval + pull-to-refresh

V1.5 (Follow-up)

SSE (Server-Sent Events) with 5-minute polling fallback
